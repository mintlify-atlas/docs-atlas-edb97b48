`room` is a single-import Go middleware for [Gin](https://github.com/gin-gonic/gin) that turns traffic spikes into an orderly waiting room. When your application hits capacity, every excess request gets a ticket, sees their queue position update in real time, and is admitted automatically the moment a slot opens — no dropped requests, no blind retries.

## Get up and running

<Steps>
  <Step title="Install">
    ```bash
    go get github.com/andreimerlescu/room
    ```
    Requires Go 1.22+ and Gin.
  </Step>
  <Step title="Initialize and register">
    ```go main.go
    package main

    import (
        "log"
        "github.com/andreimerlescu/room"
        "github.com/gin-gonic/gin"
    )

    func main() {
        r := gin.Default()

        wr := &room.WaitingRoom{}
        if err := wr.Init(500); err != nil {
            log.Fatal(err)
        }
        defer wr.Stop()

        wr.RegisterRoutes(r)

        r.GET("/", func(c *gin.Context) {
            c.String(200, "You're in!")
        })

        r.Run(":8080")
    }
    ```
    The 501st concurrent request sees the waiting room. Every earlier request passes through unmodified.
  </Step>
  <Step title="Add optional features">
    Configure skip-the-line pricing, VIP passes, lifecycle callbacks, and runtime capacity adjustments as your needs grow.
  </Step>
</Steps>

## Why room?

When your application hits capacity, you have three choices:

| Strategy | What happens | User experience |
|---|---|---|
| **Drop (429)** | Reject the request | User sees an error, retries blindly, amplifies the spike |
| **Blind queue** | Buffer with no ordering | No position awareness, no ETA, users refresh and make it worse |
| **Waiting room** | Issue a ticket, show position, admit in order | User waits calmly, knows their place, gets in automatically |

`room` implements the third strategy with FIFO ordering, live position tracking, a polished waiting-room page, and a full suite of production features.
