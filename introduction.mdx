`room` is a single-import Go middleware for [Gin](https://github.com/gin-gonic/gin) that converts traffic spikes into an orderly waiting room. When your application reaches capacity, every excess request receives a ticket, sees its queue position update in real time, and is admitted automatically the moment a slot opens — no dropped requests, no blind retries, no infrastructure changes required.

## Why a waiting room?

When your application hits capacity you have three choices. Only one of them keeps users calm and eliminates blind retries.

| Strategy | What happens | User experience |
|---|---|---|
| **Drop (429)** | Reject the request immediately | User sees an error, retries at random, amplifies the spike |
| **Blind queue** | Buffer with no ordering guarantee | No position awareness, no ETA — users refresh and make it worse |
| **Waiting room** | Issue a ticket, show live position, admit in FIFO order | User waits calmly, knows their place, gets in automatically |

`room` implements the third strategy behind a single middleware call. Your handlers never change — they see normal requests arriving at the rate you configure.

## Key features

<CardGroup cols={2}>
  <Card title="FIFO ordering" icon="arrow-down-1-9">
    Monotonically increasing ticket counter ensures every client is admitted in the exact order it arrived. No starvation, no randomness.
  </Card>
  <Card title="Live position tracking" icon="gauge">
    The built-in waiting room page polls `GET /queue/status` every 3 seconds and updates the queue position in place — no refresh required.
  </Card>
  <Card title="Zero handler changes" icon="circle-check">
    The middleware gates traffic before your handlers run. Route logic, response formats, and middleware chains stay exactly as they are.
  </Card>
  <Card title="Ghost ticket reaper" icon="ghost">
    A background goroutine evicts abandoned tokens on a configurable interval (default 5 minutes), advancing the queue past stalled positions.
  </Card>

## Installation

Install with `go get`. No configuration files, no external services, no infrastructure.

```bash
go get github.com/andreimerlescu/room
```

<Info>Requires **Go 1.22+** and [github.com/gin-gonic/gin](https://github.com/gin-gonic/gin). The only transitive dependency is [sema](https://github.com/andreimerlescu/sema), the semaphore implementation that backs the concurrent slot limit.</Info>

## Minimal setup

Three lines of setup code is all that is required. The 501st concurrent request sees the waiting room; every earlier request passes through unmodified.

```go
wr := &room.WaitingRoom{}
wr.Init(500)      // max 500 concurrent requests
defer wr.Stop()   // clean shutdown, no leaked goroutines
wr.RegisterRoutes(r)
// Every route registered after this line is gated.
```
