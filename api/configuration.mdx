All configuration methods on `WaitingRoom` are safe to call at any time, including while the server is handling traffic. Setters that accept durations or numeric ranges validate their input and return a typed error on failure. Setters that accept strings or booleans accept any value without error.

## Capacity

### `SetCap` / `Cap`

```go
func (wr *WaitingRoom) SetCap(cap int32) error
func (wr *WaitingRoom) Cap() int32
```

`SetCap` adjusts the maximum number of concurrently active requests. Expanding capacity immediately opens new semaphore slots. Shrinking capacity drains gracefully through the underlying `sema` implementation â€” existing in-flight requests complete normally, and the window narrows as slots are released.

<ParamField path="cap" type="int32" required>
  New capacity. Must be `â‰Ą 1`. Returns `ErrInvalidCap` if less than 1.
</ParamField>

```go
// Double capacity at runtime in response to autoscaler signal.
if err := wr.SetCap(wr.Cap() * 2); err != nil {
    log.Printf("SetCap failed: %v", err)
}
```

`Cap` returns the current value as last set by `Init` or `SetCap`.

<Note>
`SetCap` is safe to call concurrently. The capacity change takes effect atomically â€” `ticketReady` and `QueueDepth` computations remain consistent with the updated value.
</Note>

## Queue depth limit

### `SetMaxQueueDepth` / `MaxQueueDepth`

```go
func (wr *WaitingRoom) SetMaxQueueDepth(max int64) error
func (wr *WaitingRoom) MaxQueueDepth() int64
```

`SetMaxQueueDepth` caps the number of requests that may wait simultaneously. When the queue reaches this limit, new arrivals receive `503 Service Unavailable` immediately rather than being queued.

<ParamField path="max" type="int64" required>
  Maximum queue depth. `0` means unlimited (the default). Negative values return `ErrInvalidMaxQueueDepth`.
</ParamField>

```go
// Shed load once 10 000 requests are already waiting.
if err := wr.SetMaxQueueDepth(10_000); err != nil {
    log.Fatal(err)
}

// Remove the limit (revert to unlimited).
wr.SetMaxQueueDepth(0)
```

`MaxQueueDepth` returns the current setting. A return value of `0` means unlimited.

## Cookie settings

### `SetSecureCookie`

```go
func (wr *WaitingRoom) SetSecureCookie(secure bool)
```

Controls the `Secure` attribute on the `room_ticket` session cookie. The default is `false` so that plain-HTTP local development works without any configuration.

<Warning>
Set `secure` to `true` for any deployment served over HTTPS â€” whether directly or behind a TLS-terminating proxy such as Cloudflare, nginx, or an AWS ALB. In proxy setups, `c.Request.TLS` is often `nil` even though the end-user connection is encrypted, so the middleware cannot auto-detect TLS.
</Warning>

```go
// Call before serving any traffic.
wr.SetSecureCookie(true)
```

There is no corresponding getter. The value is stored atomically and used internally by the middleware.

### `SetCookiePath` / `CookiePath`

```go
func (wr *WaitingRoom) SetCookiePath(path string)
func (wr *WaitingRoom) CookiePath() string
```

Sets the `Path` attribute of the `room_ticket` cookie. The default is `"/"`, which scopes the cookie to all paths on the domain.

<ParamField path="path" type="string">
  Cookie path. An empty string is treated as `"/"`.
</ParamField>

Use a narrower path when multiple applications share the same domain and you want the waiting room to apply only to a specific path prefix.

```go
// Scope the cookie to the checkout flow only.
wr.SetCookiePath("/checkout")
```

---

### `SetCookieDomain` / `CookieDomain`

```go
func (wr *WaitingRoom) SetCookieDomain(domain string)
func (wr *WaitingRoom) CookieDomain() string
```

Sets the `Domain` attribute of the `room_ticket` cookie. The default is `""`, which causes the browser to use the request host (no cross-subdomain sharing).

<ParamField path="domain" type="string">
  Cookie domain. Empty string lets the browser use the request host.
</ParamField>

```go
// Share the waiting room session across subdomains.
wr.SetCookieDomain(".example.com")
```

## Reaper interval

### `SetReaperInterval` / `ReaperInterval`

```go
func (wr *WaitingRoom) SetReaperInterval(d time.Duration) error
func (wr *WaitingRoom) ReaperInterval() time.Duration
```

`SetReaperInterval` changes how frequently the background reaper scans for abandoned (ghost) tickets and evicts them. The reaper goroutine picks up the new interval immediately without a restart.

<ParamField path="d" type="time.Duration" required>
  Eviction interval. Valid range: `5s` â€“ `24h`. Returns `ErrReaperInterval` if outside this range.
</ParamField>

- **Tighter interval** (e.g., `30s`) reduces the window in which a ghost ticket stalls the queue, at the cost of more frequent lock contention on the token store.
- **Looser interval** (e.g., `30m`) is appropriate when queue depth is low and memory pressure is not a concern.

```go
// Aggressive eviction for high-throughput endpoints.
if err := wr.SetReaperInterval(30 * time.Second); err != nil {
    log.Fatal(err)
}

// Relaxed eviction for a low-traffic admin panel.
if err := wr.SetReaperInterval(15 * time.Minute); err != nil {
    log.Fatal(err)
}
```

`ReaperInterval` returns the current interval as a `time.Duration`.

## Skip-the-line pricing

Both `SetRateFunc` **and** `SetSkipURL` must be set for the skip offer to appear on the waiting room page. Configuring only one of them suppresses the offer.

### `SetRateFunc`

```go
func (wr *WaitingRoom) SetRateFunc(fn RateFunc)

// RateFunc signature:
type RateFunc func(queueDepth int64) float64
```

`SetRateFunc` configures the per-position pricing function used by `QuoteCost` and `PromoteToken`. When set, the `/queue/status` response includes `skip_cost` and `rate_per_pos` fields that the waiting room page uses to display pricing.

<ParamField path="fn" type="RateFunc">
  Pricing function. Receives the current queue depth and returns the per-position cost. Pass `nil` to disable promotions entirely.
</ParamField>

```go
// Flat rate: $1.00 per position.
wr.SetRateFunc(func(depth int64) float64 {
    return 1.00
})

// Surge pricing: rate rises with queue depth.
wr.SetRateFunc(func(depth int64) float64 {
    return 0.50 + float64(depth)*0.01
})

// Disable promotions.
wr.SetRateFunc(nil)
```

### `SetSkipURL` / `SkipURL`

```go
func (wr *WaitingRoom) SetSkipURL(url string)
func (wr *WaitingRoom) SkipURL() string
```

Sets the URL that the "Pay to skip" button on the waiting room page navigates to. This is typically a Stripe Checkout session, crypto invoice, or similar payment page hosted by your application.

<ParamField path="url" type="string">
  Absolute URL of the payment page. Empty string hides the skip offer.
</ParamField>

The handler at this URL can read the `room_ticket` cookie to identify which queued client is paying. After verifying payment, call `PromoteToken` or `PromoteTokenToFront` to move them forward in the queue.

```go
wr.SetSkipURL("https://example.com/checkout/skip")
```

`SkipURL` returns the currently configured URL as a string.

## VIP passes

### `SetPassDuration` / `PassDuration`

```go
func (wr *WaitingRoom) SetPassDuration(d time.Duration) error
func (wr *WaitingRoom) PassDuration() time.Duration
```

`SetPassDuration` configures the lifetime of VIP passes issued by `GrantPass` after a successful skip-the-line payment. A client holding a valid pass is auto-promoted to the front of the queue on any subsequent queue entry, without requiring another payment.

<ParamField path="d" type="time.Duration" required>
  Pass lifetime. `0` disables passes (single-use promotion only). When non-zero, valid range is `1m` â€“ `24h`. Returns `ErrPassDuration` if outside this range.
</ParamField>

When passes are disabled (the default, `d = 0`), `PromoteToken` still works as a single-use operation â€” the client is promoted once and must pay again if they re-enter the queue.

```go
// Issue passes valid for 2 hours after payment.
if err := wr.SetPassDuration(2 * time.Hour); err != nil {
    log.Fatal(err)
}

// Disable passes (single-use only).
wr.SetPassDuration(0)
```

`PassDuration` returns the current setting. A return value of `0` means passes are disabled.

## Custom waiting room page

### `SetHTML`

```go
func (wr *WaitingRoom) SetHTML(html []byte)
```

`SetHTML` replaces the built-in waiting room HTML page with a custom one. The replacement takes effect immediately for all subsequent queued requests; in-flight requests that already received a response are unaffected.

Pass `nil` to revert to the embedded default `waiting_room.html`.

<ParamField path="html" type="[]byte">
  Custom HTML bytes. `nil` reverts to the built-in default page.
</ParamField>

Custom HTML supports two template placeholders that the middleware substitutes on each response:

| Placeholder | Replaced with |
|---|---|
| `{{.Position}}` | The client's current queue position (integer). |
| `{{.SkipURL}}` | The skip-the-line payment URL from `SetSkipURL`. Empty string if not configured. |

```go
custom, err := os.ReadFile("my_waiting_room.html")
if err != nil {
    log.Fatal(err)
}
wr.SetHTML(custom)
```

```html
<!-- my_waiting_room.html -->
<!DOCTYPE html>
<html>
<head><title>Please wait</title></head>
<body>
  <h1>You are number {{.Position}} in line.</h1>
  {{if .SkipURL}}
    <a href="{{.SkipURL}}">Skip the line</a>
  {{end}}
</body>
</html>
```

<Note>
`SetHTML` is protected by an internal mutex and is safe to call concurrently with in-flight requests.
</Note>
